About

In this project, we programmed a Zumo with 3 Sharp IR sensors to avoid obstacles while it's on a path. Avoidance of obstacles changes the course of the Zumo.

The Sharp distance sensors are a popular choice for many projects that require accurate distance measurements. This IR sensor is more economical than sonar rangefinders, yet it provides much better performance than other IR alternatives.

Reading the sensor values

Initially I connected each sensor to a breadboard to see the values they were giving me. I noticed that the voltage and distance are inversely proportional, and the values I was getting on my serial monitor was the voltage divided by a constant 0.0049 which gave ranges of values from 0 to 1023 at 0 and 5v respectively. Although it didnt't get close to 1023, The highest value I was getting was in the range of 500's. The code for the sensor test is here

Mapping and Object avoidance

Once I saw how the sensor values are gotten, I used a map function to map those values to reference numbers for a better understanding of the distance. By using the idea behind PID I was able to form an algorithm which uses the difference in sensor values to manipulate the speed of each zumomotor using differnent if statements Zumo Object Avoidance code can be found here
